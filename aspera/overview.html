Using Aspera with Brightcove

Notes:
  • Files put on the Aspera server are kept for up to 10 days; after that they are deleted.
  • If an user has access to more than one Video Cloud account and tries to ingest content using Aspera, the content will be ingested to the first Video Cloud account the system finds for that user. To avoid this, the user must have specific credentials for each one of the Video Cloud accounts to which he/she should be able to upload content using this service.

    One workaround: You can create a new email alias using a + character if your email provider supports it (e.g. Gmail) Example:

    • Normal email address: mary@mycompany.com
    • Alias email addresses:
      • mary+sports_upload@mycompany.com
      • mary+news_upload@mycompany.com
      • mary+prod_upload@mycompany.com
      • mary+dev_upload@mycompany.com

The basic workflow for using Aspera to transfer your files to Video Cloud is as follows are shown below. Note that you will need to install the IBM Aspera Connect extension for your browser. The first time you login to Aspera, the dialog below will appear and guide you through the steps - see Installing Aspera Connect for more information.

  1. Go to https://ingest-transfer.brightcove.com/ and log in using you Video Cloud credentials with the proper permissions (see Credentials below).
  2. Upload the files you want to ingest into Video Cloud.
  3. Wait for the transfer to finish
  4. Monitor the rest of the ingestion in Video Cloud using Studio or the CMS API.

Credentials

There are two kinds of credentials for using Aspera for ingestion:

  • Upload - for simple ingestion of videos without metadata or additional assets. Simply upload the video files. If a manifest of metadata is present, it will be ignored.
    Simple upload requires special permissions - submit a Support Request to obtain these permissions.
  • Upload with Metadata - in this case, file transfer will start when the manifest file providing metadata information is uploaded. This operation can be used to add metadata and assets to a video on upload, to update existing metadata, to add assets to a video already ingested or to re-transcode or source replacement to a video in Video Cloud. See the Manifest Reference for details of this XML file. Note that there is a limit of 100 videos per manifest file.
    Upload with metadata requires special permissions - submit a Support Request to obtain these permissions.
