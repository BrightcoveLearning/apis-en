Introduction

The automated captions feature allows you to enable automatic audio transcribing to create captions for your live stream.

If you are new to the Live API, review the Create a Live Stream Using the Brightcove Live API document.

This feature can also be enabled in Video Cloud Studio. For details, see the Creating and Managing Live Events document.

Enable automated captions

To create a job with transcribe support for automated captions, add the following attributes to the regular Job creation payload:

{
  "manifest_api": {
    "enabled": true
},
"transcribe_state": "ON"
}

Set the language

You can set the language for the generated captions. In this example, the language is set to Spanish, US:

{
  "manifest_api": {
    "enabled": true
  },
  "transcribe_state": "ON",
  "transcribe_options": {
      "language_code": "es-US"
  }
}

You also have the flexibility to let the system infer the language of the ingest stream by specifying a list of probable languages.

Notes:
  • You should not provide both language_code and language_detection together. If they both are provided then language_code takes the precedence.
  • You cannot provide input languages as the same language in different dialects. For example, you cannot provide [“en-US”, “en-AU”] as an input for probable_input_languages.
{
  "manifest_api": {
    "enabled": true
  },
  "transcribe_state": "ON",
  "transcribe_options": {
      "language_detection": {
          "auto_detect_language": true,
          "probable_input_languages": ["es-US", "en-US"],
          "preferred_language": "en-US"
      }
  }
}

For a complete list of supported languages, see the Supported languages and language-specific features document.

The only languages supported are those with streaming support in the Data Input column.

Enable captions mid stream

Enabling automated captions is supported after the job has been created any time independent of the job state. New Endpoints have been defined and shall be shared shortly to accomplish the sam.

The endpoint would be:

PATCH https://<TBD>/v2/jobs/{job_id}/transcribe

  Payload:
  {
    "enabled": true/false
  }
Turning Transcription ON/OFF will not impact existing playback sessions. Players must refresh for the changes to take effect because the master manifest is not fetched once playback begins. New sessions will start with Master manifest fetch so they'll align with the current state of transcribe flag.

Limitations

Here is a list of limitations with the live automated captions feature:

  • Transcription is only supported for non low latency HLS Streams. Low Latency HLS and DASH streams don't have transcription support at this time.
  • You cannot enable Transcribe support for Jobs already created in your account.
  • Currently, we provide captions only in one language. We don't support multi language captions at the moment. The Default language for captions is set to en-US
  • The accuracy of the captions depends on the transcript result from AWS in a timely manner. We will skip captions for a segment if AWS wasn't able to deliver the same before the segment slides off the sliding window by delivering an empty VTT file.

AWS Limitations

Some limitations are specific to how Amazon transcribes speech to text. For details, see the How Amazon Transcribe works document.

  • The number of concurrent streaming transcriptions is set to 25 which can be increased by requesting a quota increase. Based on AWS TAM Inputs, the typical turnaround is ~1 week.
  • The maximum limit for audio file duration that can be sent to AWS for transcribing (both file / streaming) is capped at 4 hours. This seems to be a hard limit that can't be raised. This means that the connection will go down and be re-established automatically when the 4 hours streaming limit is exceeded, resulting in non availability of captions during that interval.
