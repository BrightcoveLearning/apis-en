What are Packaged Plugins?

Packaged Plugins are player features, including Advertising, Thumbnail Seeking, Playlists, and others, that have historically been handled by a separate service called the Plugin Registry. The Plugin Registry was designed to make managing web player plugins easy, but over the years, it has led to confusion and mismatches in versions, which in turn has led to problems that could have been avoided.

A common problem has been trying to use an old plugin version with a new player version (or vice versa). The plugin relies on specific player functionality that may be different than expected or not present, leading to problems or failures. This was a big problem with advertising as browsers moved to block autoplay with sound by default.

By moving features into the player's core, we ensure that the plugin is the correct version for the player because there is no longer a separate version. It's a part of the player tested to function as expected.

How do I configure my Packaged Plugins?

To use them in the studio, see this guide.

To use them via the Player Management API, see this guide.

What if I need to use plugin version X with player version Y?

You do not. The concept of Packaged Plugins is to ensure that the feature you are using is 100% compatible with your player version. Trying to use a different version goes against this.

How do I know if I'm using a Packaged Plugin?

The main difference is in the player configuration JSON, you'll see ”is_packaged”: true.

Here is the Plugin Registry JSON structure:

    
          "plugins": [{
            "injected_version": "2.0.10",
            "name": "customEndscreen",
            "registry_id": "@brightcove/videojs-custom-endscreen",
            "version": "2.x"
          }],
          "scripts": [
            "//players.brightcove.net/videojs-custom-endscreen/2.0.10/videojs-custom-endscreen.min.js"
          ],
          "stylesheets": [
            "//players.brightcove.net/videojs-custom-endscreen/2.0.10/videojs-custom-endscreen.css"
          ],

Here is the new Packaged Plugins JSON structure:

        
            "plugins": [{
                "injected_version": "2.0.10",
                "name": "customEndscreen",
                "is_packaged": true
              }],
              {
                "name": "thumbnails",
                "is_packaged": true
              },

In the player information modal, you'll see the plugin version as “Packaged with Player.” If you see a version string, this would be an older, legacy version, which can be updated with the current packaged version instead.

If you have Registry Plugins and upgrade your player to v7.21.0 or above, any of the eligible plugins will be converted automatically to Packaged Plugins.
If you have Packaged Plugins and downgrade your player below v7.21.0, any of the packaged plugins on your players will be converted back into registry plugins.

What if I must “sideload” a core plugin during player load time?

You should never need to do that. Brightcove's player management system is designed to compile all the features you need and deliver a working player with those features to your users.

For example, instead of sideloading the IMA3 plugin, if you need to deliver ads to your users, use a player with the IMA3 feature enabled. If you don’t need ads in other situations, use a different player without the IMA3 feature enabled, or configure ads at runtime in a very similar way than you would have with sideloading — except that you no longer will need to include the plugin separately.

What about custom or third-party plugins?

We have not changed or removed any functionality for those. These changes only affect plugins managed by the plugin registry.
